Ship your game as a physical object.

Relic is an open format for taking a video game outside digital stores without giving up updates, DLC, and content control. This page explains how it works inside.

Flash architecture

Three partitions, three roles.

The memory embedded in the statue's base is split into three independent spaces. Each has write rules, encryption, and access consistent with what it holds.

CorePartition 01

The game files. Read-only via hardware write-protect. Always visible, always copyable. Encryption for the launcher is optional; ownership never is. The user can archive, copy, or run the files without going through us.

read-onlyhw-protectedarchivable
FluxPartition 02

Hosts Ember, the embedded app. Optional autorun, zero installation. If the system already has the companion app Forge installed, Ember delegates and exits — no duplication.

autorunzero-installportable
DriftPartition 03

DLC, patches, extra content. Writable only by authenticated apps. Every update is signed server-side, applied locally. The studio keeps full control over what lands on the chip.

signedwritableversioned
Identity and license

The chip is the license.

Every memory has a unique serial number. The access key for encrypted content is derived locally from the UID via HKDF. No account, no connection required on first boot, no validation server.

The physical Relic is the sole license title. Move it, and the license moves with it. Lose it, and you lose it. It's deliberately symmetric with the physical objects everyone already understands.

Local derivation

HKDF from the chip UID. No server communication needed to play.

Zero account

No signup, no login. The statue is your access token.

Offline-first

The game boots without a network. Network is only used for DLC and updates.

Preservation

Even without us, Core files remain accessible to the user.

Distribution

Limited runs, curated, physical.

We produce in house: modelling, SLA printing, manual post-processing. Every piece is numbered. The target is indie studios who want an unconventional physical edition of their game — sold as a collectible, not as a substitute for the digital channel.

We handle statue design, production, packaging, and software integration. The studio provides the game and defines the theme. Print runs and royalties are agreed title by title.
